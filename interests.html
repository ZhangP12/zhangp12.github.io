Favorite Mathematical Problem: Does P = NP?

The P versus NP problem is probably the most important problem in theoretical computer science, and ranks among the top in all areas of mathematics. It is one of the seven Millennium Problems, with one million US dollars given to the person(s) who solves one of those problems. To date, only the Poincaré conjecture has been solved.
Informally, the P versus NP problem asks whether every problem whose solution that can be quickly verified can also be solved quickly.

A Longer Explanation

The P versus NP problem concerns how quickly an algorithm can solve a problem given input n. P stands for polynomial time, while NP stands for nondeterministic polynomial time.
A P problem requires at worst xk time to solve, where x is "length" of the problem, while k is an integer. Hence the name polynomial time. NP problems ARE NOT the hard problems I was talking about. They're any problem whose solution can be verified quickly, which obviously includes P problems. NP-hard problems are, well, hard. They require kx time to solve (sometimes x!), so time grows exponentially as the length of the problem grows. That's really bad.
